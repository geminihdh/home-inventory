# 내 손안의 우리집 물건 관리 (Home Inventory PWA) - 디자인 명세서

이 문서는 사용자의 휴대폰 브라우저에 데이터를 저장하고 관리하는 개인용 물건 관리 웹 앱(PWA)의 설계를 정의합니다.

## 1. 개요
- **목적:** 집안의 물건을 카테고리별로 등록하여 위치, 유통기한 등을 한눈에 파악.
- **주요 가치:** 
  - **프라이버시:** 모든 데이터는 사용자의 휴대폰 내부에만 저장 (서버 없음).
  - **편의성:** 앱 설치 없이 브라우저에서 '홈 화면에 추가'하여 즉시 사용.
  - **직관성:** 리스트와 썸네일 기반의 빠른 검색 및 관리.

## 2. 기술 스택
- **Frontend:** React (TypeScript)
- **Styling:** Vanilla CSS (또는 사용자 요구에 따른 현대적인 스타일링)
- **Data Storage:** IndexedDB (브라우저 로컬 데이터베이스)
- **PWA:** Service Workers, Web App Manifest (오프라인 구동 및 홈 화면 설치 지원)
- **Hosting:** GitHub Pages (무료 정적 호스팅)

## 3. 데이터 모델 (Schema)
`Item` 인터페이스:
- `id`: string (UUID)
- `name`: string (물건명)
- `image`: string (Base64 형식의 압축된 이미지 데이터)
- `description`: string (내용/설명)
- `location`: string (위치 - '안방', '주방' 등)
- `purchaseDate`: string (구매시기, YYYY-MM-DD)
- `expiryDate`: string (유통기한, YYYY-MM-DD)
- `memo`: string (비고)
- `createdAt`: number (생성일시 타임스탬프)

## 4. 주요 화면 및 기능

### 4.1 메인 목록 화면
- **물건 리스트:** 썸네일, 이름, 위치, 유통기한 D-Day 표시.
- **검색 바:** 이름 및 위치 필터링.
- **정렬:** 기본적으로 최신순 또는 유통기한 임박순 정렬.
- **추가 버튼:** Floating Action Button (FAB)을 통한 등록 화면 진입.

### 4.2 물건 등록/수정 화면
- **사진 촬영/업로드:** 카메라 연동 및 이미지 리사이징(용량 최적화).
- **입력 폼:** 이름, 위치(자주 쓰는 위치 선택 포함), 구매일, 유통기한, 메모 필드.
- **위치 관리:** '자주 쓰는 위치'를 미리 등록하고 선택 가능.

### 4.3 상세 정보 팝업/화면
- 물건의 모든 정보를 크게 확인하고 수정/삭제 가능.

## 5. 비기능적 요구사항
- **성능:** 이미지 데이터를 효율적으로 압축하여 IndexedDB 용량 최적화.
- **오프라인 지원:** 인터넷 연결 없이도 데이터 조회 및 등록 가능.
- **사용자 경험:** 모바일 환경에 최적화된 터치 UI/UX.

## 6. 보안 및 데이터 정책
- 사용자의 모든 데이터는 브라우저 외부로 절대 전송되지 않음.
- 별도의 회원가입이나 로그인 절차가 없음.
